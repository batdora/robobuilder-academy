# Learning from Demonstrations with GMMs

When a human demonstrates a task multiple times, the demonstrations are never identical -- there is natural variation. **Gaussian Mixture Models (GMMs)** provide a probabilistic framework to encode this variation, capturing both the mean trajectory and the acceptable range of deviation.

## The LfD Pipeline

The full Learning from Demonstration pipeline consists of:

1. **Human demonstrator** provides multiple demonstrations $X$
2. **Dimensionality reduction** (optional, e.g., PCA) produces $\xi$
3. **Probabilistic encoding** with GMM produces parameters $\hat{\xi}_s, \hat{\Sigma}_s$
4. **Task constraints** are determined via a metric $P$ on the encoded data
5. **Optimal trajectory generation** via GMR produces $\xi'$
6. **Reconstruction** maps back to data space $X'$
7. **Robot imitator** executes the trajectory

## Gaussian Mixture Models (GMM)

A GMM models the probability density of demonstration data as a weighted sum of $K$ Gaussian components:

$$
P(\boldsymbol{\xi}) = \sum_{k=1}^{K} \pi_k \, \mathcal{N}(\boldsymbol{\xi} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

where:
- $\pi_k$ are **mixing coefficients** (prior probabilities), with $\sum_k \pi_k = 1$ and $\pi_k \geq 0$
- $\boldsymbol{\mu}_k$ are the **mean vectors** of each component
- $\boldsymbol{\Sigma}_k$ are the **covariance matrices** of each component
- $K$ is the number of Gaussian components

Each Gaussian component:

$$
\mathcal{N}(\boldsymbol{\xi} \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k) = \frac{1}{(2\pi)^{d/2} |\boldsymbol{\Sigma}_k|^{1/2}} \exp\left(-\frac{1}{2}(\boldsymbol{\xi} - \boldsymbol{\mu}_k)^T \boldsymbol{\Sigma}_k^{-1} (\boldsymbol{\xi} - \boldsymbol{\mu}_k)\right)
$$

## Expectation-Maximization (EM) Algorithm

GMM parameters $\{\pi_k, \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k\}$ are learned via the **EM algorithm**, which iteratively:

### E-step: Compute Responsibilities

For each data point $\boldsymbol{\xi}_n$ and component $k$, compute the posterior probability (responsibility):

$$
\gamma_{nk} = \frac{\pi_k \, \mathcal{N}(\boldsymbol{\xi}_n \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_{j=1}^{K} \pi_j \, \mathcal{N}(\boldsymbol{\xi}_n \mid \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}
$$

This is the probability that data point $n$ was generated by component $k$.

### M-step: Update Parameters

Using the responsibilities, update the parameters:

$$
N_k = \sum_{n=1}^{N} \gamma_{nk} \quad \text{(effective number of points in component } k\text{)}
$$

$$
\boldsymbol{\mu}_k^{\text{new}} = \frac{1}{N_k} \sum_{n=1}^{N} \gamma_{nk} \boldsymbol{\xi}_n
$$

$$
\boldsymbol{\Sigma}_k^{\text{new}} = \frac{1}{N_k} \sum_{n=1}^{N} \gamma_{nk} (\boldsymbol{\xi}_n - \boldsymbol{\mu}_k^{\text{new}})(\boldsymbol{\xi}_n - \boldsymbol{\mu}_k^{\text{new}})^T
$$

$$
\pi_k^{\text{new}} = \frac{N_k}{N}
$$

### Convergence

EM is guaranteed to increase the log-likelihood at each iteration:

$$
\mathcal{L} = \sum_{n=1}^{N} \log P(\boldsymbol{\xi}_n) = \sum_{n=1}^{N} \log \sum_{k=1}^{K} \pi_k \mathcal{N}(\boldsymbol{\xi}_n \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

EM converges to a **local** maximum (not necessarily global). Initialization matters -- common choices include k-means initialization.

## Gaussian Mixture Regression (GMR)

Once we have a GMM encoding of the demonstrations, we use **GMR** to generate a smooth trajectory.

Given the joint distribution over time $t$ and trajectory values $\boldsymbol{\xi}_s$, we partition each Gaussian:

$$
\boldsymbol{\mu}_k = \begin{bmatrix} \mu_k^t \\ \boldsymbol{\mu}_k^s \end{bmatrix}, \quad
\boldsymbol{\Sigma}_k = \begin{bmatrix} \Sigma_k^{tt} & \Sigma_k^{ts} \\ \Sigma_k^{st} & \Sigma_k^{ss} \end{bmatrix}
$$

For a query time $t^*$, the conditional distribution is:

**Responsibilities at query point:**

$$
h_k(t^*) = \frac{\pi_k \mathcal{N}(t^* \mid \mu_k^t, \Sigma_k^{tt})}{\sum_{j=1}^{K} \pi_j \mathcal{N}(t^* \mid \mu_j^t, \Sigma_j^{tt})}
$$

**Conditional mean and covariance for each component:**

$$
\hat{\boldsymbol{\mu}}_k^s(t^*) = \boldsymbol{\mu}_k^s + \Sigma_k^{st}(\Sigma_k^{tt})^{-1}(t^* - \mu_k^t)
$$

$$
\hat{\boldsymbol{\Sigma}}_k^s = \Sigma_k^{ss} - \Sigma_k^{st}(\Sigma_k^{tt})^{-1}\Sigma_k^{ts}
$$

**Mixture of conditional distributions:**

$$
\hat{\boldsymbol{\mu}}^s(t^*) = \sum_{k=1}^{K} h_k(t^*) \hat{\boldsymbol{\mu}}_k^s(t^*)
$$

$$
\hat{\boldsymbol{\Sigma}}^s(t^*) = \sum_{k=1}^{K} h_k(t^*) \left[\hat{\boldsymbol{\Sigma}}_k^s + \hat{\boldsymbol{\mu}}_k^s(t^*)(\hat{\boldsymbol{\mu}}_k^s(t^*))^T\right] - \hat{\boldsymbol{\mu}}^s(t^*)(\hat{\boldsymbol{\mu}}^s(t^*))^T
$$

The result is a smooth trajectory with uncertainty estimates at each time step.

## Task-Parameterized GMM (TP-GMM)

Standard GMM encodes demonstrations in a fixed frame. **TP-GMM** generalizes by learning in **multiple reference frames** simultaneously.

Given $P$ task parameters (frames), each defined by $\{\mathbf{A}_p, \mathbf{b}_p\}$ (rotation and translation):

Each Gaussian component is defined in each frame $p$:

$$
\boldsymbol{\mu}_{k,p}, \quad \boldsymbol{\Sigma}_{k,p}
$$

At execution time with new frame parameters $\{\mathbf{A}_p^*, \mathbf{b}_p^*\}$, the components are transformed and combined using the product of Gaussians:

$$
\mathcal{N}(\hat{\boldsymbol{\mu}}_k, \hat{\boldsymbol{\Sigma}}_k) \propto \prod_{p=1}^{P} \mathcal{N}(\mathbf{A}_p^* \boldsymbol{\mu}_{k,p} + \mathbf{b}_p^*, \mathbf{A}_p^* \boldsymbol{\Sigma}_{k,p} (\mathbf{A}_p^*)^T)
$$

This enables the robot to **generalize** the learned skill to new spatial configurations (e.g., grasping an object at a different position).

## Worked Example: Simple 1D GMM

Given 2 demonstrations of a 1D trajectory at times $t = [0, 0.5, 1.0]$:
- Demo 1: $[0, 3, 5]$
- Demo 2: $[0, 2.5, 5.5]$

With $K = 2$ Gaussians (after EM converges):
- Component 1 ($\pi_1 = 0.5$): $\mu_1 = [0.25, 2.75]$, capturing the start
- Component 2 ($\pi_2 = 0.5$): $\mu_2 = [0.75, 5.25]$, capturing the end

GMR at $t^* = 0.6$: The regression produces a weighted combination, yielding approximately $\hat{\mu}^s \approx 3.8$ with some variance reflecting the demonstration variability.

## Key Takeaway

> GMMs provide a probabilistic encoding of multiple demonstrations, capturing both the average trajectory and the natural variation. EM learns the GMM parameters iteratively, and GMR produces smooth trajectories with uncertainty estimates. TP-GMM extends this to multiple reference frames, enabling generalization to new spatial configurations -- a crucial capability for flexible robot imitation.
