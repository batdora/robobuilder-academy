# Q-Learning

Q-learning is one of the most important breakthroughs in reinforcement learning. It is an **off-policy** TD control algorithm that directly learns the optimal action-value function $Q^*$, regardless of the policy being followed. This makes it remarkably powerful and simple.

## Q-Learning: Off-Policy TD Control

### The Q-Learning Update Rule

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[r_{t+1} + \gamma \max_a Q(s_{t+1}, a) - Q(s_t, a_t)\right]
$$

Compare this carefully with SARSA:

| | SARSA | Q-Learning |
|---|-------|------------|
| **Update** | $Q(s,a) + \alpha[r + \gamma Q(s', a') - Q(s,a)]$ | $Q(s,a) + \alpha[r + \gamma \max_{a'} Q(s', a') - Q(s,a)]$ |
| **Next action in target** | $a'$ = action actually taken | $\max_{a'}$ = best possible action |
| **Policy type** | On-policy | Off-policy |

The critical difference: Q-learning uses $\max_a Q(s_{t+1}, a)$ in its target instead of $Q(s_{t+1}, a_{t+1})$. This means the update is based on the **best possible action** from the next state, not the action the agent actually takes.

### The TD Target and TD Error

**Q-learning TD target:**

$$
\text{target} = r_{t+1} + \gamma \max_a Q(s_{t+1}, a)
$$

**Q-learning TD error:**

$$
\delta_t = r_{t+1} + \gamma \max_a Q(s_{t+1}, a) - Q(s_t, a_t)
$$

### Backup Diagram

The Q-learning backup diagram shows a filled circle (state-action pair) at the top, branching to open circles (next states), and then at each next state, taking the **max** over all next actions (shown by an arc). This contrasts with SARSA's backup, which follows a single path to one next action.

## Algorithm: Q-Learning

```
Initialize Q(s, a) arbitrarily
Repeat (for each episode):
    Initialize s
    Repeat (for each step of episode):
        Choose a from s using policy derived from Q (e.g., epsilon-greedy)
        Take action a, observe r, s'
        Q(s, a) <- Q(s, a) + alpha * [r + gamma * max_a' Q(s', a') - Q(s, a)]
        s <- s';
    until s is terminal
```

### Key Differences from SARSA's Algorithm

Notice that Q-learning:
1. **Does not choose $a'$ before updating** -- the update uses $\max_{a'} Q(s', a')$, not a specific chosen action
2. **Does not need** $a \leftarrow a'$ -- the next action is chosen fresh from the new state $s'$ at the start of the next iteration
3. The **behavior policy** (used to select actions, e.g., $\varepsilon$-greedy) can be different from the **target policy** (greedy, embedded in the max)

## On-Policy vs Off-Policy

### On-Policy (SARSA)

The agent learns the value of the policy **it is actually following**:
- If the agent explores (takes a random action due to $\varepsilon$-greedy), this exploration affects the Q-value updates
- The learned Q-values reflect the $\varepsilon$-greedy policy, not the greedy policy
- The policy being evaluated and the policy being improved are the **same**

### Off-Policy (Q-Learning)

The agent learns the value of the **optimal policy** regardless of which policy it follows:
- The max in the update means Q-learning always targets $Q^*$
- Even if the agent explores randomly, the Q-values converge to the optimal values
- The **behavior policy** (how actions are selected) differs from the **target policy** (what is being learned about)

**Why is off-policy important?**
- Can learn from data generated by any policy (even a random one)
- Can learn about the optimal policy while following an exploratory policy
- Can learn from observing other agents or from historical data

## Convergence of Q-Learning

Q-learning converges to $Q^*$ under the following conditions:

1. **All state-action pairs** continue to be visited (explored) -- this is ensured by $\varepsilon$-greedy with $\varepsilon > 0$
2. The **learning rate** $\alpha$ satisfies:
   - $\sum_{t=0}^{\infty} \alpha_t = \infty$ (the steps are large enough to overcome initial conditions)
   - $\sum_{t=0}^{\infty} \alpha_t^2 < \infty$ (the steps decrease fast enough to converge)

In practice, a constant small $\alpha$ (like 0.1) works well, though technically this means Q-values oscillate around the optimum rather than converging exactly.

## Epsilon-Greedy Exploration in Q-Learning

Q-learning typically uses $\varepsilon$-greedy exploration:

$$
a_t = \begin{cases} \arg\max_a Q(s_t, a) & \text{with probability } 1 - \varepsilon \\ \text{random action} & \text{with probability } \varepsilon \end{cases}
$$

- $\varepsilon$ controls the exploration rate
- Common choice: $\varepsilon = 0.1$ (explore 10% of the time)
- Can **decay** $\varepsilon$ over time: start with more exploration, gradually shift to exploitation
- Even with exploration, Q-learning converges to $Q^*$ (the optimal Q-values under the greedy policy)

## The Cliff Walking Example

This is the classic example that illustrates the difference between SARSA and Q-learning.

### Setup
- A 4x12 gridworld
- Start state **S** at bottom-left, Goal state **G** at bottom-right
- The bottom row between S and G is **"The Cliff"**
- Stepping into the cliff: reward = **-100** and agent is sent back to S
- All other transitions: reward = **-1**
- Undiscounted ($\gamma = 1$), $\varepsilon = 0.1$

### Two Paths

- **Optimal path:** Along the bottom row, right next to the cliff (shortest path, but risky with $\varepsilon$-greedy)
- **Safe path:** Goes up, across the top, and down (longer, but avoids cliff)

### Q-Learning's Behavior

Q-learning learns the **optimal** Q-values, which correspond to the policy of walking along the cliff edge (the shortest path). However, because the agent uses $\varepsilon$-greedy to select actions, it **occasionally falls off the cliff** during training. The Q-values converge to optimal, but the agent's **online performance** suffers due to exploration.

The reward per episode for Q-learning fluctuates around -50 or lower, with occasional large drops to -100+ when the agent falls off the cliff.

### SARSA's Behavior

SARSA learns Q-values that account for the $\varepsilon$-greedy exploration. Since the agent occasionally takes random actions, SARSA "knows" that walking next to the cliff is dangerous (random actions could push it off). Therefore, SARSA learns to take the **safe path** along the top.

The reward per episode for SARSA converges to around -25 (the safe path length), which is **better online performance** than Q-learning.

### The Paradox

- **Q-learning** finds the optimal policy but performs worse during training
- **SARSA** finds a safer (suboptimal) policy but performs better during training
- If $\varepsilon$ were gradually reduced to 0, both would converge to the optimal policy
- The difference is fundamental to on-policy vs off-policy learning

## Q-Learning vs SARSA: Complete Comparison

| Feature | Q-Learning | SARSA |
|---------|-----------|-------|
| **Type** | Off-policy | On-policy |
| **Target** | $r + \gamma \max_{a'} Q(s', a')$ | $r + \gamma Q(s', a')$ |
| **Learns** | Optimal $Q^*$ | $Q^\pi$ for current policy |
| **Behavior policy** | Can be anything | Must match learned policy |
| **Exploration impact** | Not reflected in Q-values | Reflected in Q-values |
| **Cliff walking** | Optimal but risky path | Safe but longer path |
| **Online performance** | Worse (with exploration) | Better (accounts for exploration) |
| **Asymptotic policy** | Optimal | Optimal (if $\varepsilon \to 0$) |

## Worked Example: One-Step Q-Learning Update

Consider a simple scenario:
- Current state: $s = A$, action taken: $a = \text{right}$
- Reward received: $r = -1$
- Next state: $s' = B$
- Current Q-values at $B$: $Q(B, \text{up}) = 3$, $Q(B, \text{down}) = 1$, $Q(B, \text{right}) = 5$, $Q(B, \text{left}) = 2$
- Current $Q(A, \text{right}) = 2$
- $\alpha = 0.1$, $\gamma = 0.9$

**Q-learning update:**

$$
Q(A, \text{right}) \leftarrow 2 + 0.1 \times [-1 + 0.9 \times \max(3, 1, 5, 2) - 2]
$$

$$
= 2 + 0.1 \times [-1 + 0.9 \times 5 - 2]
$$

$$
= 2 + 0.1 \times [-1 + 4.5 - 2]
$$

$$
= 2 + 0.1 \times 1.5 = 2.15
$$

**SARSA update** (suppose $a' = \text{up}$ was actually chosen):

$$
Q(A, \text{right}) \leftarrow 2 + 0.1 \times [-1 + 0.9 \times 3 - 2]
$$

$$
= 2 + 0.1 \times [-1 + 2.7 - 2]
$$

$$
= 2 + 0.1 \times (-0.3) = 1.97
$$

Notice Q-learning gives a higher updated value (2.15 vs 1.97) because it uses the max Q-value at the next state, while SARSA uses the Q-value of the action actually selected.

## Summary: The Complete Picture

We have now covered the full spectrum of tabular RL methods:

| Method | Model? | Bootstraps? | Sample-based? | Update |
|--------|--------|-------------|---------------|--------|
| **DP (Policy/Value Iteration)** | Yes | Yes | No | $\max_a \sum_{s'} P_{ss'}^a[R_{ss'}^a + \gamma V(s')]$ |
| **Monte Carlo** | No | No | Yes | $V(s) + \alpha[R_t - V(s)]$ |
| **TD(0)** | No | Yes | Yes | $V(s) + \alpha[r + \gamma V(s') - V(s)]$ |
| **SARSA** | No | Yes | Yes | $Q(s,a) + \alpha[r + \gamma Q(s',a') - Q(s,a)]$ |
| **Q-Learning** | No | Yes | Yes | $Q(s,a) + \alpha[r + \gamma \max_{a'} Q(s',a') - Q(s,a)]$ |

Each method represents a different point in the trade-off space between computation, data, and model requirements.

## Key Takeaway

> Q-learning is the crown jewel of tabular RL: a simple, model-free algorithm that converges to optimal Q-values regardless of exploration policy. The single difference from SARSA -- using max instead of the actual next action -- makes it off-policy and enables it to learn the true optimal values. Understanding when to use SARSA (safer online performance) vs Q-learning (optimal convergence) is essential for any RL practitioner.
